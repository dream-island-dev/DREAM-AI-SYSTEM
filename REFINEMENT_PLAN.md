## 🔴 P0 — תקינות וחלקות (לתקן קודם)

### P0.1 — חוסר התאמה בטקסונומיית המחלקות (באג אמיתי)
יש **שלוש אוצרות-מילים שונות למחלקות** במערכת, והן לא תואמות:
- `App.js` (דשבורד/משמרות/צ'קליסט) משתמש בעברית: `קבלה, ניקיון, מסעדה, תחזוקה, ביטחון, ספא`
- מסד הנתונים (`profiles.department` CHECK) מתיר רק אנגלית: `housekeeping, maintenance, reception, spa, management`
- `DataUpload.js` / `ShiftGenerator.js` משתמשים במפתחות אנגלית (`DEPT_LABEL`)

**למה זה שובר:** מנהל ש"מחלקתו" = `קבלה` לא יתאים לשורות שמתויגות `reception`; בידול המחלקות (RLS/סינון) נשבר; הצ'קליסט מסנן לפי עברית בזמן שהאורחים/משמרות נשמרים באנגלית. בנוסף `מסעדה` ו`ביטחון` **כלל לא קיימים** ברשימת ה-DB.
**תיקון:** להגדיר טקסונומיה קנונית אחת (מומלץ: מפתחות אנגלית במסד) + מפת תוויות-תצוגה אחת (`DEPT_LABELS`) בקובץ משותף, ולהשתמש בה בכל מקום. להרחיב את ה-CHECK ב-DB לכל המחלקות בפועל.

### P0.2 — קריאת ה-chat Edge Function ללא header אימות
`AgentChat.send()` קורא ל-`CHAT_EDGE_URL` עם `fetch` **בלי** `Authorization`/`apikey`. אם הפונקציה לא נפרסה עם `--no-verify-jwt`, זה יחזיר 401. לשם השוואה — `DataUpload`/`ShiftGenerator` משתמשים ב-`supabase.functions.invoke()` שמוסיף אוטומטית את ה-anon key וה-JWT.
**תיקון:** להחליף את ה-`fetch` הגולמי ב-`supabase.functions.invoke("chat", { body })` — אחיד, מטפל ב-auth וב-CORS, ופחות שביר.

### P0.3 — טקסט "מצב דמו" מיושן ומטעה
`AgentChat` מציג למשתמש להגדיר `REACT_APP_BACKEND_URL` (ה-backend הישן של Apps Script), בזמן שהמערכת עברה ל-Supabase Edge Functions. הלוגיקה `USE_DEMO_MODE` כמעט אף פעם לא נכונה ב-production.
**תיקון:** לעדכן את הקופי וההיגיון לבדוק `CHAT_EDGE_URL` בלבד; להסיר התייחסויות ל-`REACT_APP_BACKEND_URL`.

### P0.4 — מקור-אמת כפול: יצירת ה-System Prompt
יש שני מסלולים ליצירת הפרומפט: `buildSystemPrompt()` בצד-לקוח (בשאלון) ו-`generate-agent-profile` Edge Function (מסומן "לא בשימוש פעיל"). שניהם עלולים לסטות.
**תיקון:** לבחור מסלול אחד אמין (מומלץ: ה-Edge Function עם grounding מה-memory), ולמחוק/להקפיא את השני.

### P0.5 — דאטה דמה מול דאטה אמיתי בעמודים התפעוליים
`App.js` עדיין מכיל מערכי דמה גדולים (`initialEmployees/Shifts/Calls/Checklists`), בזמן ש-`DataUpload`/`ShiftGenerator` כותבים ל-Supabase. חלק מהעמודים כבר עברו ל-`useTable` (קריאה מ-DB), חלק אולי לא.
**תיקון:** לוודא שכל עמוד תפעולי קורא מ-Supabase, ולמחוק את הדמה. אחרת המשתמש רואה נתונים שמתאפסים ב-refresh.

## 🟡 P1 — נוחות חוויית-משתמש (הליטוש המורגש ביותר)

### P1.1 — מערכת Toast + Confirm אחידה (הניצחון הכי גדול)
כרגע יש שלושה דפוסי משוב: Toast יפה (DataUpload/ShiftGenerator), `window.confirm`/`alert` נייטיב (AdminPanel — צורם מול ה-UI המלוטש), ומשוב inline (AgentChat).
**תיקון:** רכיב `Toast` + `ConfirmDialog` משותף אחד, בשימוש בכל האפליקציה. מיידית מרגיש מקצועי ואחיד.

### P1.2 — נגישות בסיסית (כרגע: אפס)
סריקה מצאה **0 תכונות `aria-`/`role`/`tabIndex`** בכל הקוד. כפתורים הם אימוג'י בלבד (👍/👎/✕), ו-`div`-ים לחיצים (אזור גרירה, תיבות-סימון) לא נגישים למקלדת.
**תיקון:** `aria-label` לכל כפתור-אייקון, `role="button"`+`tabIndex`+מקלדת ל-div-ים לחיצים, ו-`:focus-visible` נראה. גם מסייע ל-SEO ולמובייל.

### P1.3 — Error Boundary גלובלי
שגיאת render אחת מפילה את כל האפליקציה ל"מסך לבן".
**תיקון:** React Error Boundary עם מסך-נפילה ידידותי בעברית + כפתור "טען מחדש".

### P1.4 — רינדור Markdown בצ'אט
תשובות הסוכן מוצגות כטקסט גולמי (`pre-wrap`), אבל Gemini/Claude מחזירים Markdown (**מודגש**, רשימות, כותרות). זה נראה "שבור".
**תיקון:** רינדור Markdown בטוח בבועות התשובה (ספרייה קלה / parser מינימלי). שדרוג קריאוּת עצום.

### P1.5 — מצבי טעינה וריקות עקביים
לצ'אט יש מצבי טעינה טובים; לחלק מהשליפות (עובדים ב-ShiftGenerator) אין מצב טעינה/ריק. משתמש על חיבור איטי רואה מסך ריק בלי הסבר.
**תיקון:** skeletons/מצבי-ריק אחידים לכל שליפת Supabase.

### P1.6 — מחלקות לא ריאקטיביות
שינוי מחלקות ב-AdminPanel דורש רענון דף (`localStorage['di_departments']` נקרא רק ב-load).
**תיקון:** להעביר מחלקות ל-React state/Context שמתעדכן מיידית, או טעינה מ-DB.

## 🟢 P2 — ליטוש ועמידוּת

- **P2.1 — לוגים על catch שקטים:** הרבה `catch {}` בולעים שגיאות. להוסיף `console.warn` לפחות, כדי שאפשר יהיה לדבג production.
- **P2.2 — מזהי הודעות:** `Date.now()` עלול להתנגש בלחיצות מהירות. לעבור ל-counter/uuid.
